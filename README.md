# LÉR — Langage Écriture-Réalité

**Système sémiotique et performatif pour analyser, transformer et co-créer du sens symbolique.**

> *"Le texte devient acte — l'acte devient signe — le signe devient monde."*

---

## Qu'est-ce que le LÉR ?

Le Langage Écriture-Réalité (LÉR) est un langage hybride **déclaratif et performatif** conçu
pour activer une relation consciente entre l'expression, la perception et la transformation du réel.

Il repose sur une intuition radicale : le langage ne décrit pas le réel — il le **façonne**.
Écrire est un acte, pas une description.

Né entre 2012 et 2014 dans un corpus de discussions sur le rapport entre langage, pensée et réalité,
le LÉR a été formalisé en un système opératoire complet articulant philosophie sémiotique,
grammaire symbolique et architecture computationnelle.

---

## Fondements philosophiques

Le LÉR s'ancre dans cinq traditions de pensée :

| Référence | Apport |
|---|---|
| **Julia Kristeva** | Le sujet en procès, le polylogue |
| **Henri Lefebvre** | La transduction, l'espace du possible |
| **Douglas Hofstadter** | Récursivité de la conscience |
| **Michel Serres** | Médiation et circulation des signes |
| **Paul Virilio** | L'accident comme révélateur symbolique |

## Les 4 piliers

| Pilier | Fonction |
|---|---|
| **Résonance** | Détection des correspondances, motifs répétitifs, synchronicités |
| **Transduction** | Conversion d'un registre expressif en un autre |
| **Métalangage** | Le texte se lit, se commente et se corrige lui-même |
| **Polylogue** | Circulation de la parole entre auteur, lecteur, IA et monde |

## Les 5 instructions fondamentales

Le LÉR opère via cinq instructions qui forment une **grammaire minimale mais complète** :

| Instruction | Rôle | Métaphore |
|---|---|---|
| `META(texte)` | Analyse des couches de sens | La boussole |
| `RESONE(s1, s2…)` | Détection de motifs et archétypes | La carte |
| `TRANSDUIT(source:type) → cible:type` | Conversion inter-registres | Le traducteur |
| `SIGNIFIE("geste symbole")` | Acte symbolique performatif | Le levier |
| `POLYLOGUE(input)` | Co-création évolutive | L'interlocuteur |

**Exemple de chaînage :**
```
RESONE("miroir", "ombre")
  → arch("errant")
  → META("errant")               → {dynamique: transition}
  → SIGNIFIE("nommer errant")    → ouverture("nouveau cycle")
```

## Domaines d'application

- **Analyse symbolique** — rêves, textes, événements réels
- **Transformation personnelle** — blocages, états intérieurs, rituels d'écriture
- **Narration performative** — récits vivants, co-création, jeux narratifs
- **Intelligence artificielle** — agent IA symbolique, GPS du sens
- **Géopolitique (LÉR-Géo)** — lecture symbolique d'événements mondiaux, détection de signaux faibles, scénarios fractals

## Structure du dépôt

```
ler-skill/
├── README.md                  ← ce fichier
├── SKILL.md                   ← entrée principale (frontmatter Claude + architecture)
└── references/
    ├── instructions.md        ← 5 instructions détaillées avec exemples commentés
    ├── lexique.md             ← symboles, archétypes, états intérieurs, zones
    └── sequences.md           ← scripts, protocoles, LÉR-Géo, algorithme agent
```

### `SKILL.md`
Fichier d'entrée principal. Contient le frontmatter de déclenchement pour les systèmes Claude,
l'architecture générale, les 4 piliers, les 5 instructions en résumé, et les renvois vers
les fichiers de référence.

### `references/instructions.md`
Documentation complète de chaque instruction : syntaxe, types, sorties, exemples commentés,
règles de chaînage et techniques fondamentales.

### `references/lexique.md`
Table des symboles opératoires, les 12 archétypes fondamentaux avec leurs cycles,
les états intérieurs codifiés, les zones symboliques et la structure des cycles archétypaux.

### `references/sequences.md`
Scripts opératoires (S1–S6), protocoles d'usage (analyse, rêve, prédiction, transformation,
navigation), scripts de transformation symbolique (T1–T5), scripts prédictifs (P1–P4),
application LÉR-Géo complète avec exemple, et format JSON standardisé `LER_Packet`.

## Utilisation comme Skill Claude

Ce dépôt est structuré comme un **skill Claude** (progressive disclosure) :

1. Le frontmatter de `SKILL.md` sert de déclencheur — Claude consulte le skill dès qu'un
   utilisateur mentionne : archétype, synchronicité, motif, seuil, ombre, transduction,
   écriture-réalité, acte symbolique, ou les 5 instructions par leur nom.
2. `SKILL.md` est chargé en contexte à chaque déclenchement.
3. Les fichiers `references/` sont chargés **à la demande**, selon le besoin précis
   (analyse d'une instruction spécifique, consultation du lexique, exécution d'un protocole).

## Principes opératoires

**Minimalité expressive** — peu de mots, beaucoup d'effets. Chaque terme est lourd de sens,
chaque instruction a plusieurs niveaux de lecture simultanés.

**Dualité sens / action** — chaque instruction produit simultanément un effet sémantique
(modifie la carte symbolique) et un effet performatif (agit sur la dynamique narrative).

**Responsabilité symbolique** — le LÉR est un langage qui agit. Son usage exige respect,
intention et conscience. Les analyses produites sont symboliques et narratives,
jamais des prédictions certaines.
